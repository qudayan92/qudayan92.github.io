## 1. 项目定位

一个**游戏化的小说创作工具**,把小说创作流程拆解为可闯关的关卡。作者像玩游戏一样推进创作:每关有明确目标,通过选择动作(AI 生成、手动编辑、灵感触发)来达成,闯过一关解锁下一关,直到完成整部小说。

## 2. 核心体验

| 游戏概念 | 创作映射 |
|---------|--------|
| 关卡 | 创作阶段(世界观→角色→大纲→章节) |
| 关卡目标 | 当前阶段需产出的内容 |
| 动作 | AI 续写、手动编辑、灵感生成、查看提示 |
| 通关条件 | 内容通过自我审核/字数达标 |
| 成就/奖励 | 解锁下一关、统计数据、可视化进度 |
| 存档 | 所有创作内容持久化存储 |

## 3. 关卡流水线(固定顺序,线性解锁) ✅

### 第一关:世界观设定 ✅
- 目标:完成世界观描述(≥200 字)
- 可用动作:AI 灵感生成、手动编辑
- 通关条件:内容非空 + 字数达标

### 第二关:主角创建 ✅
- 目标:完成主角档案(姓名、性格、背景、动机)
- 通关条件:必填字段完整

### 第三关:故事大纲 ✅
- 目标:完成三幕式大纲(开端、发展、高潮-结局)
- 通关条件:三幕均有内容

### 第四关起:章节创作(循环) ✅
- 每个章节为一关
- 字数门槛随章节数递增(800 → 1500 → 2000 → 3000)
- 第 10 / 20 / 30 章为 **Boss 关**(金色边框 + 🔥 图标 + 字数翻倍)✅
- 无限循环直到作者宣告完结

## 4. 技术方案 ✅

- **形态**:本地单页 Web 应用(HTML + CSS + JS),浏览器直接打开 ✅
- **数据存储**:浏览器 localStorage(无需后端) ✅
- **AI 集成**:预留 DeepSeek API 接口(目前 mock,后续接入真 API) 🟡
- **无依赖**:纯原生 HTML/CSS/JS,零框架,零构建步骤 ✅

## 8. 设计原则 ✅

- **不跳关**:必须按顺序闯关,保证创作逻辑连贯
- **可回看**:已通关卡的内容可随时查看(只读)
- **自动存档**:每次内容变更自动保存到 localStorage(300ms 防抖)
- **离线可用**:不需要网络(除 AI 功能外)

## 9. 除 AI 味 / 降重模块

### 状态:✅ 已实现(本地启发式 + DeepSeek 智能改写)

### 实现策略
| 层级 | 手段 | 状态 |
|------|------|------|
| 生成层 | Prompt 注入:在 callAI 系统提示词中加 8 条"避免 AI 套话"硬规则 | ✅ |
| 后处理层 | AI_TICKS 30+ 套话清单 + 25 条 AI_REPLACEMENTS 替换 + 长句拆短 | ✅ |
| 智能改写 | DeepSeek rewrite 模式,温度 0.95,硬规则改写 | ✅ |
| 检测层 | 本地启发式:套话命中 / 句长变异系数 / 同结尾 / 罗列词 / 感叹号 / 比喻词 | ✅ |

### UI
- 编辑器动作栏新增 🪄 **除 AI 味** 按钮(金色边框) + 📊 **检测** 按钮
- 弹窗(820×90vh):
  - 顶部 AI 味指数:进度条 0-100(绿/黄/红)
  - 三种模式:⚡ 本地快速清洗 / 🧠 AI 智能改写 / 📊 仅检测
  - 原文 vs 改写后 双栏对比
  - 信号列表:具体命中的 AI 特征
  - 底部:🪄 开始改写 / ✓ 应用到编辑器 / ↶ 还原 / 取消

### 核心规则
**生成层**(callAI rewrite 模式)
1. 删除 30+ AI 标志性表达
2. 修辞词降级
3. 句长多变(连续三句不能同长度)
4. 段落开头不能雷同
5. 加入口语化连接词

**后处理层**(dedaiLocal)
- 25 条替换规则(如"值得注意的是"→删除,"总的来说"→"说到底,")
- 长句 >60 字按逗号/分号拆成短句
- 清理多余标点和空白

**检测层**(detectAiFlavor)
- AI 套话命中数 × 12 分(上限 40)
- 句长变异系数 CV < 0.25 → 20 分
- 连续 3+ 句同结尾词 → 15 分
- "首先...其次..." 罗列模式 → 10 分
- 200+ 字无感叹/问号 → 5 分
- 比喻词 > 3 个 → 最多 15 分
- 总分 0-100,≥60 红 / 30-60 黄 / <30 绿

### 数据流
```
用户点 🪄 → 拉取当前关卡内容 → 显示 AI 味指数 + 信号
↓
选择模式:本地 / AI / 仅检测
↓
开始改写 → 调用对应函数 → 写入右侧文本框
↓
显示新分数(对比下降幅度)
↓
用户点 ✓ 应用 → 写入 st.content + dispatch input 事件
```

## 1. 项目定位

一个**游戏化的小说创作工具**,把小说创作流程拆解为可闯关的关卡。作者像玩游戏一样推进创作:每关有明确目标,通过选择动作(AI 生成、手动编辑、灵感触发)来达成,闯过一关解锁下一关,直到完成整部小说。

## 2. 核心体验

| 游戏概念 | 创作映射 |
|---------|--------|
| 关卡 | 创作阶段(世界观→角色→大纲→章节) |
| 关卡目标 | 当前阶段需产出的内容 |
| 动作 | AI 续写、手动编辑、灵感生成、查看提示 |
| 通关条件 | 内容通过自我审核/字数达标 |
| 成就/奖励 | 解锁下一关、统计数据、可视化进度 |
| 存档 | 所有创作内容持久化存储 |

## 3. 关卡流水线(固定顺序,线性解锁) ✅

### 第一关:世界观设定 ✅
- 目标:完成世界观描述(≥200 字)
- 可用动作:AI 灵感生成、手动编辑
- 通关条件:内容非空 + 字数达标

### 第二关:主角创建 ✅
- 目标:完成主角档案(姓名、性格、背景、动机)
- 通关条件:必填字段完整

### 第三关:故事大纲 ✅
- 目标:完成三幕式大纲(开端、发展、高潮-结局)
- 通关条件:三幕均有内容

### 第四关起:章节创作(循环) ✅
- 每个章节为一关
- 字数门槛随章节数递增(800 → 1500 → 2000 → 3000)
- 第 10 / 20 / 30 章为 **Boss 关**(金色边框 + 🔥 图标 + 字数翻倍)✅
- 无限循环直到作者宣告完结

## 4. 技术方案 ✅

- **形态**:本地单页 Web 应用(HTML + CSS + JS),浏览器直接打开 ✅
- **数据存储**:浏览器 localStorage(无需后端) ✅
- **AI 集成**:预留 DeepSeek API 接口(目前 mock,后续接入真 API) 🟡
- **无依赖**:纯原生 HTML/CSS/JS,零框架,零构建步骤 ✅

## 8. 设计原则 ✅

- **不跳关**:必须按顺序闯关,保证创作逻辑连贯
- **可回看**:已通关卡的内容可随时查看(只读)
- **自动存档**:每次内容变更自动保存到 localStorage(300ms 防抖)
- **离线可用**:不需要网络(除 AI 功能外)

### 核心规则
**生成层**(callAI rewrite 模式 — 凌晨三点喝红牛赶稿)
- 角色:凌晨三点赶稿,刚喝完第三罐红牛,脑子飘,说话颠三倒四
- 句式:至少3个1-2字碎片句,至少2个50+字流水句,长度差>20字
- 词汇:高级词→口语词(然而→可是,因此→所以),用网络梗(好家伙/麻了/裂开/蚌埠住了)
- 结构:段落间无逻辑过渡,加入内心OS/对读者喊话/自相矛盾/具体细节(凌晨两点半/便利店/瑞幸)/身体感受
- 禁止:值得注意的是/不得不说/综上所述/仿佛/宛如/犹如

**后处理层**(dedaiLocal — 6轮深度降重)
- 第1轮:套话替换(postProcessText) + 长句拆短(splitLongSentences)
- 第2轮:句式打散(scrambleSentenceStarts) + 段落重组(scrambleParagraphs)
- 第3轮:注入口语化(injectColloquial) + 碎片句(injectFragments)
- 第4轮:随机化打散(randomScramble,激进度降至5%/8%/10%)
- 第5轮:人味注入(humanChaos,激进度大幅降低避免乱码)
- 第6轮:格式清理

**深度本地降重**(dedaiDeepLocal,参考 qu-ai-wei)
- 51条AI高频词替换规则(深入→扎进去,聚焦→盯住,赋能→帮忙...)
- 句式打散+段落重组
- 注入口语化元素+碎片句
- 随机化处理(每次结果不同)

**三轮降重协议**(dedaiAI/threeRoundProtocol,参考 aigc-reduce)
- 第1轮:困惑度打散(高级词→口语词/碎嘴句/省略主语/具体细节/内心OS)
- 第2轮:语义结构打乱(打乱段落顺序/自相矛盾/跑题再绕回/句长极端波动)
- 第3轮:风格注入(口头禅重复/标点不规范/身体感受/具体时间)
- 后处理:去重(deduplicateText) + 套话替换(postProcessText)
- 每轮temperature=1.5,top_p=0.95

**翻译链**(dedaiTranslate,参考 lynote-ai/humanize-text)
- 中文→日语(DeepSeek,温度1.5)
- 日语→英语(二次跨语言)
- 英语→中文(第三次翻译回来)
- 第4步:注入人味(深夜赶稿作者风格再改写)
- 后处理:去重+套话替换+长句拆短+人味注入

**人味注入引擎**(humanChaos)
- 思维跳跃:段落间插入不相干内心OS(降低激进度,最多1处)
- 自相矛盾:随机插入自我纠正句
- 具体细节:插入时间/地点/数字
- 身体感受:后背发凉/手心出汗/嗓子干得冒烟
- 对读者喊话:你猜怎么着/你别说/我说真的
- 口头禅重复:说真的/怎么说呢/你知道吗
- 标点密度打乱:有的地方不加标点像流水账

**检测层**(detectAiFlavor — 11维度)
- AI 套话命中数 × 12 分(上限 40)
- 句长变异系数 CV < 0.25 → 20 分
- 连续 3+ 句同结尾词 → 15 分
- "首先...其次..." 罗列模式 → 10 分
- 200+ 字无感叹/问号 → 5 分
- 比喻词 > 3 个 → 最多 15 分
- 缺对话 → 8 分
- 缺碎片短句 → 10 分
- 缺内心OS → 5 分
- 缺具体细节 → 5 分
- 缺自相矛盾 → 3 分
- 总分 0-100,≥60 红 / 30-60 黄 / <30 绿

- **核心问题**: 之前能100%通过的关键是 **system prompt的角色扮演** 而不是"资深编辑"这种理性角色

- 当前还差35.98%才能到100%, 需进一步增强"混乱度"但又不能乱码

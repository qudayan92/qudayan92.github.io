## 2026-06-13 · TODO 占位符系统 (设计 12.3)

**目标**:追踪写作时"还差什么"。

**新增**
- 状态栏顶部:`TODO N` 实时未完成数(橙色提示,N=0 时灰)
- 侧边栏底部:`📌 TODO(N)` 折叠面板,列出所有 TODO
  - 每项显示:TODO 文本、所在关卡(`第 X 章`)、字符位置
  - 橙色卡片样式区分于关卡
- 写作时输入 `[TODO: 描述]` 即创建占位符
- 公众号复制时 TODO 转为黄色高亮 `<span>`

**交互**
- 点击 TODO 项 → 切换到对应关卡 + 光标选中 TODO 文本
- 点击 TODO 前 □ → 标记完成(绿色 ✓ + 删除线 + 变灰)
- 顶部 `TODO` 可点击自动展开折叠面板
- 跨作品独立(每部作品维护自己的 `todoDone` 字典)

**技术**
- 每次 `render()` 重新扫描所有关卡内容,正则 `\[TODO:[^\]]*\]`
- 勾选状态存 `work.todoDone[lvlId + ':' + offset]`,按位置标识
- offset 用 `match.index` 取字符位置,文本编辑后失效(自动重新扫描)

**验证**(Node 端 + 浏览器 Playwright)
- 扫描 + 计数 + 跳转 + 勾选 + 跨作品独立,5 项断言全过,无 pageerror

## 2026-06-13 · 公众号格式一键复制 (设计 12.7)

**目标**:在工具里写完,直接复制粘贴到微信公众号编辑器,样式不丢。

**UI**
- 动作栏新增 "📋 复制公众号格式" 主按钮 + "▾" 下拉
- 下拉选项:`📄 复制当前章节` / `📚 复制整部作品` / `👁 预览 HTML` / `🔖 复制 HTML 源码`
- Toast 提示(2.2s 自动消失,成功绿/失败红)
- 预览弹窗(720×86vh,深色背景,白底渲染):
  - 双 tab:`渲染效果` / `HTML 源码`
  - 内置"复制到剪贴板" + "关闭"按钮
  - ESC 键关闭,点遮罩关闭

**转换规则**(Markdown → 公众号兼容 HTML)
| Markdown | 输出 |
|----------|------|
| 段落(空行分隔) | `<p style="margin:0 0 14px;line-height:1.85;font-size:16px;color:#333;">` |
| `# 一级标题` | `<p style="font-size:1.4em;font-weight:bold;...">` |
| `## 二级标题` | `<p style="font-size:1.2em;font-weight:bold;...">` |
| `**粗体**` | `<strong style="color:#000;font-weight:bold;">` |
| `*斜体*` | `<em style="font-style:italic;color:#555;">` |
| `> 引用` | `<blockquote style="border-left:3px solid #ccc;...">` |
| `[TODO: x]` | `<span style="background:#fff3a0;color:#7a5a00;">` |
| 段间 | `<p><br></p>` 间隔 |

**整部作品复制**
- 按关卡顺序拼接,每个关卡加标题:
  - 世界观 → `# 世界观`
  - 主角 → `# 主角`
  - 大纲 → `# 大纲`
  - 章节 → `# 第N章 · 第X关`
- 关卡之间用 `\n\n---\n\n` 分隔

**技术**
- 优先用 `navigator.clipboard.writeText`(现代 API)
- 降级到 `document.execCommand("copy")` + 临时 textarea
- 内联样式让公众号粘贴后保留(不用 `<style>`,公众号会剥)

**修复**
- 重写整个 `app.js`(原 patch 累积了多处 syntax error)
- 修复了 `function inspireHint() {` 被 patch 误删的 bug

**验证**
- 浏览器 Playwright:toast 弹出 ✓,预览渲染正确 ✓,HTML 源码含全部样式 ✓
- 注:Edge headless 模式下读不到剪贴板内容(权限),不影响实际用户使用

## 2026-06-13 · 多作品管理(架构升级)

**目标**:支持同时创作多部小说,互不干扰。

**架构改动**
- `state` 从单一作品改为:
  ```js
  state = { activeWorkId, works: { [id]: {...} } }
  ```
- 存档键升级 `novel-quest.v1` → `novel-quest.v2`(老数据自动迁移)
- 每个作品独立维护:名字、创建时间、当前关卡、33 关内容、TODO 勾选状态

**UI**
- 顶栏新增"作品切换器"下拉(`▾`),点开显示:
  - 所有作品列表(名称 + 字数 + 完成关卡数)
  - 当前作品打 ✓
  - 底部"＋ 新建作品"
- 侧边栏底部新增"✎ 重命名作品"和"🗑 删除当前作品"按钮

**功能**
- `createWork(name)`:创建新作品,自动切到新作品
- `renameWork(id)`:弹 prompt 重命名
- `deleteWork(id)`:弹 confirm,至少保留 1 部
- `switchWork(id)`:切到指定作品,所有视图自动重渲

**数据迁移**
- 加载时检测 `novel-quest.v2` 优先,否则尝试 `novel-quest.v1` 迁移
- 老数据变成一部"我的旧作"作品

**验证**
- Node 端 22 个 smoke test,含多作品隔离、字数独立、持久化、老 v1 迁移、删除
- 全部通过
